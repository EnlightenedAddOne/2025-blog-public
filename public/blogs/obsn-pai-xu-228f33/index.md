# 第八章：排序 (Sort)

## 1. 插入排序

### 1.1 直接插入排序

- 对应视频：`8.2_1_插入排序`
- **算法思想**：将序列分为已排序和未排序两部分，每次从未排序部分取一个元素插入到已排序部分的正确位置。
- **复杂度**：最好 $O(n)$（已排序），平均/最坏 $O(n^2)$。
- **空间**：$O(1)$。
- **稳定性**：稳定。

**C语言代码实现**：

```c
// 直接插入排序（带哨兵）
void InsertSort(ElemType A[], int n) {
    int i, j;
    for(i = 2; i <= n; i++) {           // 依次将A[2]~A[n]插入前面已排序序列
        if(A[i] < A[i-1]) {             // 若A[i]关键码小于前驱
            A[0] = A[i];                // 复制为哨兵，A[0]不存放元素
            for(j = i-1; A[0] < A[j]; --j) // 从后往前查找插入位置
                A[j+1] = A[j];          // 向后挪位
            A[j+1] = A[0];              // 复制到插入位置
        }
    }
}
```

### 1.2 希尔排序 (Shell Sort)

- 对应视频：`8.2_2_希尔排序`
- **算法思想**：又称"缩小增量排序"。先将待排序表分割成若干子表，对各子表分别进行直接插入排序，缩小增量 $d$，重复上述过程直到 $d=1$。
- **复杂度**：平均 $O(n^{1.3})$，最坏 $O(n^2)$。
- **空间**：$O(1)$。
- **稳定性**：**不稳定**。

**C语言代码实现**：

```c
// 希尔排序
void ShellSort(ElemType A[], int n) {
    int d, i, j;
    for(d = n/2; d >= 1; d = d/2) {     // 步长变化
        for(i = d+1; i <= n; ++i) {     // 需将A[i]插入有序增量子表
            if(A[i] < A[i-d]) {
                A[0] = A[i];            // 暂存在A[0]
                for(j = i-d; j > 0 && A[0] < A[j]; j -= d)
                    A[j+d] = A[j];      // 记录后移
                A[j+d] = A[0];          // 插入
            }
        }
    }
}
```

## 2. 交换排序

### 2.1 冒泡排序

- 对应视频：`8.3_1_冒泡排序`
- **算法思想**：从后往前（或从前往后）两两比较相邻元素，若为逆序则交换，直到序列比较完。每一趟冒泡能确定一个元素的最终位置。
- **复杂度**：最好 $O(n)$（有序，加flag优化），平均/最坏 $O(n^2)$。
- **空间**：$O(1)$。
- **稳定性**：稳定。

**C语言代码实现**：

```c
// 冒泡排序（带flag优化）
void BubbleSort(ElemType A[], int n) {
    for(int i = 0; i < n-1; i++) {
        bool flag = false;                  // 表示本趟是否发生交换
        for(int j = n-1; j > i; j--) {      // 一趟冒泡过程
            if(A[j-1] > A[j]) {             // 若为逆序
                swap(A[j-1], A[j]);         // 交换
                flag = true;
            }
        }
        if(flag == false) return;           // 本趟未发生交换，已有序
    }
}
```

### 2.2 快速排序 (Quick Sort) —— 考研重点

- 对应视频：`8.3_2_快速排序`
- **算法思想**：基于分治法。选一个枢轴 `pivot`，将序列划分为两部分，左边小于枢轴，右边大于枢轴，然后递归处理两个子序列。
- **复杂度**：最好/平均 $O(n \log n)$，最坏 $O(n^2)$（有序时退化）。
- **空间**：$O(\log n)$（递归栈），最坏 $O(n)$。
- **稳定性**：**不稳定**。
- **特点**：内部排序中平均性能最优。

**C语言代码实现**：

```c
// 划分函数（核心，必须手写）
int Partition(ElemType A[], int low, int high) {
    ElemType pivot = A[low];            // 将当前表中第一个元素设为枢轴
    while(low < high) {
        while(low < high && A[high] >= pivot) --high;
        A[low] = A[high];               // 将比枢轴小的元素移动到左端
        while(low < high && A[low] <= pivot) ++low;
        A[high] = A[low];               // 将比枢轴大的元素移动到右端
    }
    A[low] = pivot;                     // 枢轴元素存放到最终位置
    return low;                         // 返回存放枢轴的最终位置
}

// 快排主函数
void QuickSort(ElemType A[], int low, int high) {
    if(low < high) {
        int pivotpos = Partition(A, low, high); // 划分
        QuickSort(A, low, pivotpos-1);          // 递归排序左子表
        QuickSort(A, pivotpos+1, high);         // 递归排序右子表
    }
}
```

## 3. 选择排序

### 3.1 简单选择排序

- 对应视频：`8.4_1_简单选择排序`
- **算法思想**：每一趟在待排序元素中选取关键字最小的元素，作为有序子序列的第 $i$ 个元素，直到第 $n-1$ 趟完成。
- **复杂度**：无论有序无序，比较次数恒为 $O(n^2)$。
- **空间**：$O(1)$。
- **稳定性**：**不稳定**。
- **特点**：**移动次数最少**（最多 $3(n-1)$ 次）。

**C语言代码实现**：

```c
// 简单选择排序
void SelectSort(ElemType A[], int n) {
    for(int i = 0; i < n-1; i++) {          // 一共进行n-1趟
        int min = i;                        // 记录最小元素位置
        for(int j = i+1; j < n; j++)        // 在A[i...n-1]中选择最小元素
            if(A[j] < A[min]) min = j;      // 更新最小元素位置
        if(min != i) swap(A[i], A[min]);    // 封装的swap函数共移动3次
    }
}
```

### 3.2 堆排序 (Heap Sort)

- 对应视频：`8.4_2_堆排序`
- **算法思想**：利用完全二叉树的性质。将序列视为完全二叉树，构建"大顶堆"（根 >= 左右孩子）。建堆后，输出堆顶（与堆底交换），然后调整剩余元素为堆。
- **复杂度**：建堆 $O(n)$，排序 $O(n \log n)$，总计 $O(n \log n)$。
- **空间**：$O(1)$。
- **稳定性**：**不稳定**。

**C语言代码实现**：

```c
// 将以k为根的子树调整为大根堆
void HeadAdjust(ElemType A[], int k, int len) {
    A[0] = A[k];                            // A[0]暂存子树的根结点
    for(int i = 2*k; i <= len; i *= 2) {    // 沿key较大的子结点向下筛选
        if(i < len && A[i] < A[i+1]) i++;   // 取key较大的子结点下标
        if(A[0] >= A[i]) break;             // 筛选结束
        else {
            A[k] = A[i];                    // 将A[i]调整到双亲结点上
            k = i;                          // 修改k值，继续向下筛选
        }
    }
    A[k] = A[0];                            // 被筛选结点放入最终位置
}

// 建立大根堆
void BuildMaxHeap(ElemType A[], int len) {
    for(int i = len/2; i > 0; i--)          // 从i=[n/2]~1，反复调整堆
        HeadAdjust(A, i, len);
}

// 堆排序算法
void HeapSort(ElemType A[], int len) {
    BuildMaxHeap(A, len);                   // 初始建堆
    for(int i = len; i > 1; i--) {          // n-1趟的交换和建堆过程
        swap(A[i], A[1]);                   // 输出堆顶元素（和堆底交换）
        HeadAdjust(A, 1, i-1);              // 调整剩余i-1个元素为堆
    }
}
```

## 4. 归并排序 (Merge Sort)

- 对应视频：`8.5_1_归并排序`
- **算法思想**：将两个或多个有序表组合成一个新的有序表。2-路归并最常见：递归地将序列平分为两半，保持左右有序，然后将两个有序子序列"归并"为一个。
- **复杂度**：$O(n \log n)$。
- **空间**：$O(n)$（需要辅助数组）。
- **稳定性**：**稳定**。

**C语言代码实现**：

```c
ElemType *B = (ElemType *)malloc((n+1) * sizeof(ElemType)); // 辅助数组

// Merge: 将前后相邻的两个有序表归并为一个有序表
void Merge(ElemType A[], int low, int mid, int high) {
    int i, j, k;
    for(k = low; k <= high; k++)
        B[k] = A[k];                        // 将A中所有元素复制到B中

    for(i = low, j = mid+1, k = i; i <= mid && j <= high; k++) {
        if(B[i] <= B[j])                    // 比较B左右两段中的元素
            A[k] = B[i++];                  // 将较小值复制到A中
        else
            A[k] = B[j++];
    }
    // 若还有剩余，直接复制
    while(i <= mid)  A[k++] = B[i++];
    while(j <= high) A[k++] = B[j++];
}

// MergeSort: 归并排序
void MergeSort(ElemType A[], int low, int high) {
    if(low < high) {
        int mid = (low + high) / 2;         // 从中间划分
        MergeSort(A, low, mid);             // 对左半部分归并排序
        MergeSort(A, mid+1, high);          // 对右半部分归并排序
        Merge(A, low, mid, high);           // 归并
    }
}
```

## 5. 基数排序 (Radix Sort)

**算法思想**：不基于"比较"和"移动"，而是基于"关键字各位"的大小。对于十进制整数，从低位到高位（个位→十位→百位...），依次按该位的值将元素分配到 0~9 共 10 个桶（队列）中，再按桶的顺序收集。重复 $d$ 次（$d$ 为最大关键字的位数）。

- **复杂度**：$O(d(n+r))$。其中 $d$ 为位数（趟数），$n$ 为元素个数，$r$ 为基数（如十进制 $r=10$）。
- **空间**：$O(r)$（需要 $r$ 个队列）。
- **稳定性**：**稳定**。
- **适用性**：顺序存储和链式存储。
- **特点**：效率高，但需要额外空间，适合关键字位数较少的情况。

**C语言代码实现**（链式基数排序）：

```c
// 假设关键字为整数，使用链式队列
typedef struct LinkNode {
    ElemType data;
    struct LinkNode *next;
} LinkNode;

// 基数排序（最低位优先 LSD）
void RadixSort(LinkNode *&L, int d) { // d 为关键字位数
    LinkNode *queue[10], *tail[10];   // 10个队列（桶）
    int i, j, k;

    for(i = 0; i < d; i++) {          // 从低位到高位，共d趟
        // 初始化队列
        for(j = 0; j < 10; j++) queue[j] = tail[j] = NULL;

        // 分配：按第i位的值将节点分配到对应队列
        LinkNode *p = L->next;
        while(p != NULL) {
            k = GetDigit(p->data, i); // 取第i位的值（0~9）
            if(queue[k] == NULL) {
                queue[k] = tail[k] = p;
            } else {
                tail[k]->next = p;
                tail[k] = p;
            }
            p = p->next;
        }

        // 收集：按队列0~9的顺序，将所有节点串起来
        LinkNode *head = L;
        for(j = 0; j < 10; j++) {
            if(queue[j] != NULL) {
                head->next = queue[j];
                head = tail[j];
            }
        }
        head->next = NULL;
    }
}

// 辅助函数：获取整数x的第i位（从0开始，0为个位）
int GetDigit(int x, int i) {
    int divisor = 1;
    for(int j = 0; j < i; j++) divisor *= 10;
    return (x / divisor) % 10;
}
```

**考点提示**：

- 基数排序是**唯一不基于比较**的内部排序。
- 关键字位数 $d$ 对效率影响很大，适合位数少的场景（如学号、日期）。
- 链式基数排序比顺序基数排序更节省空间（不需要预先分配大数组）。

## 6. 各种排序比较（必背表格）

| 算法         | 时间复杂度(平均) | 时间(最坏)    | 空间        | 稳定性   | 特点             |
| :----------- | :--------------- | :------------ | :---------- | :------- | :--------------- |
| **直接插入** | $O(n^2)$         | $O(n^2)$      | $O(1)$      | 稳定     | 适合基本有序     |
| **希尔**     | $O(n^{1.3})$     | $O(n^2)$      | $O(1)$      | 不稳     | 步长             |
| **冒泡**     | $O(n^2)$         | $O(n^2)$      | $O(1)$      | 稳定     | 交换             |
| **快排**     | $O(n \log n)$    | $O(n^2)$      | $O(\log n)$ | **不稳** | **综合性能最好** |
| **简单选择** | $O(n^2)$         | $O(n^2)$      | $O(1)$      | **不稳** | 移动次数少       |
| **堆排序**   | $O(n \log n)$    | $O(n \log n)$ | $O(1)$      | **不稳** | 适合 Top K       |
| **归并**     | $O(n \log n)$    | $O(n \log n)$ | $O(n)$      | 稳定     | 需辅助空间       |
| **基数**     | $O(d(n+r))$      | $O(d(n+r))$   | $O(r)$      | 稳定     | 不比较           |

## 7. 重要考点与易错点

### 7.1 各算法易错点

- **直接插入排序**：注意哨兵 `A[0]` 的作用是暂存和避免越界，下标从 1 开始。
- **希尔排序**：增量序列影响复杂度，通常用 $d_k = n/2^k$。不稳定的原因是相同元素可能分在不同子表。
- **冒泡排序**：加入 `flag` 优化可将最好情况降至 $O(n)$。
- **快速排序**：
  - 最坏情况发生在**已排序**或**逆序**序列上（退化为冒泡）。
  - 优化：三数取中法选枢轴、随机化枢轴、尾递归优化。
- **简单选择排序**：**移动次数最少**（最多 $3(n-1)$ 次），但比较次数固定 $O(n^2)$。
- **堆排序**：
  - 建堆从 `n/2` 开始（最后一个非叶节点）。
  - 大顶堆用于升序，小顶堆用于降序。
- **归并排序**：需要 $O(n)$ 辅助空间，但可以改为**就地归并**（较复杂，不常考）。
- **基数排序**：链式比顺序节省空间，但实现复杂。

### 7.2 选择题常考陷阱

1. **稳定性**：问"哪个不稳定"→快希选堆；问"哪个稳定"→插冒归基。
2. **时间复杂度**：注意**最好、平均、最坏**的区别。
3. **空间复杂度**：
   - 快排递归栈：平均 $O(\log n)$，最坏 $O(n)$。
   - 归并需要 $O(n)$ 辅助数组。
4. **移动次数**：简单选择排序移动次数最少（最多 $3(n-1)$）。
5. **适用性**：希尔排序只能用于**顺序存储**（因为需要下标跳跃）。

### 7.3 记忆口诀

- **稳定性口诀**：**“快希选堆”不稳定**（快排、希尔、选择、堆排）。
  - 助记：_快些选一堆朋友（不稳重）_。
- **时间复杂度口诀**：
  - **快些归队** ($O(n \log n)$)：快排、希尔(优)、归并、堆排。
  - 其余基本是 $O(n^2)$。
- **关于快排**：
  - **平均性能最好**。
  - 最坏情况（有序）退化为冒泡 $O(n^2)$。
  - 空间复杂度 $O(\log n)$（递归栈），最坏 $O(n)$。
