## 第一章：嵌入式系统概述

> [!abstract] 核心定义
>
> 嵌入式系统：以应用为中心，以计算机技术为基础，软件、硬件可剪裁，适应应用系统对功能、可靠性、成本、体积和功耗严格要求的专用计算机系统。

### 1. 嵌入式系统三要素

- **嵌入性**：嵌入到对象体系中。
- **专用性**：面向特定应用场景。
- **计算机系统**：本质是计算机。

### 2. 系统组成结构

1. **硬件层**：微处理器、存储器、I/O接口。
2. **中间层 (HAL/BSP)**：硬件抽象层，实现上层系统软件与底层硬件的分离，使系统的底层驱动与硬件无关。
3. **软件层**：嵌入式操作系统、文件系统、图形用户接口、网络系统、通用组件模块。
4. **功能层**：具体应用程序，面向被空对象和用户。

### 3. 嵌入式的发展阶段

- 第一个阶段是以*单芯片为核心的可编程控制器*形式的系统
- 第二个阶段是以*嵌入式CPU为基础、以简单操作系统为核心*的嵌入式系统
- 第三个阶段是以*嵌入式操作系统*为标志的嵌入式系统
- 第四个阶段是以 _Internet_ 为标志的嵌入式系统

### 4. 嵌入式处理器

嵌入式系统至少包含一个主处理器，作为中心控制设备，主处理器的复杂性通常决定这将其归为微处理器还是微控制器。

- **微处理器（EMCU）**：微处理器的基础是通用计算机中的CPU。在应用中，将微处理器装配在专门设计的电路板上，只保留和嵌入式应用有关的母板功能。
- **微控制器（MCU）**：典型代表是单片机

### 5. 嵌入式操作系统

嵌入式操作系统是一种用途广泛的系统软件，负责嵌入式系统的全部软、硬件资源分配、调度工作，控制协调并发活动。

### 6. 嵌入式系统特点

- 嵌入式系统产业是不可垄断的高度分散的工业
- 嵌入式系统面向于特定产品和具体的用户及领域
- 嵌入式系统对软件有高要求
- 嵌入式系统开发需要特定的开发环境和开发工具
- 嵌入式系统软件需要实时操作系统

### 7. 嵌入式系统的发展趋势

1. 小型化、智能化、网络化、信息化
2. 嵌入式软件开发平台化、标准化、系统可升级、代码可重复将更受重视
3. 低功耗、低成本

## 第二章：ARM体系结构

### 1. ARM 技术特征

- **架构**：RISC（精简指令集）。
- **指令长度**：ARM指令（32位）、Thumb指令（16位）。
- **寄存器**：大量使用寄存器，寻址方式灵活。

### 2. 处理器模式 (7种)

| **模式名称** | **缩写** | **描述**                      |
| ------------ | -------- | ----------------------------- |
| 用户模式     | `usr`    | 正常执行程序                  |
| 系统模式     | `sys`    | 运行特权级操作系统任务        |
| 快速中断     | `FIQ`    | 对应高速中断处理              |
| 外部中断     | `IRQ`    | 对应普通中断处理              |
| 管理模式     | `svc`    | 操作系统保护模式（复位或SWI） |
| 中止模式     | `abt`    | 存取异常时进入                |
| 未定义模式   | `und`    | 执行未定义指令时进入          |

### 3. 寄存器组织

- **通用寄存器**：31个。
  - `R13 (SP)`：堆栈指针。
  - `R14 (LR)`：链接寄存器（保存子程序返回地址）。
  - `R15 (PC)`：程序计数器。

- **状态寄存器**：6个（CPSR 和 SPSR）。
  - **CPSR 标志位**：`N`(负)、`Z`(零)、`C`(进位)、`V`(溢出)。
    CPSR包括条件标志位、中断禁止位、当前处理器模式标志位、控制和状态位。

### 4. ARM 处理器内核命名规则

**ARM{x}{y}{z}{T}{D}{M}{I}{E}{J}{F}{-S}**

- **x**：系列代号（如 7, 9, 10, 11）。
- **y**：存储管理/保护单元。
- **z**：内含Cache 。
- T：支持 16 位 Thumb 指令集
- D：支持 JTAG 调试
- M：增强型长乘法器 (Fast Multiplier)
- I：带有嵌入式追踪宏单元ETM，用来设置断点和观察点的调试硬件
- E：增强型 DSP 指令
- J：支持 Java 加速技术
- F：具备硬件浮点运算单元
- S：可综合版本

### 5. 哈佛结构和普林斯顿结构

1. **普林斯顿结构 (Princeton Architecture)**：程序指令和数据都存放在同一个存储器中。ARM7系列基于普林斯顿结构。
2. **哈佛结构 (Harvard Architecture)**：拥有独立的程序存储器和数据存储器，并配有独立的指令总线和数据总线，处理器可以同时读取指令和存取数据，极大地提高了执行效率。ARM9系列之后基本都为哈佛结构。

### 6. 流水线技术

**概念：** 流水线是将一个重复的时序过程分解为若干个子过程，每个子过程都可有效地在其专用功能段上与其他子过程同时执行。

- ARM7 三级流水线 (3-Stage Pipeline)
  1.  **取指 (Fetch)**：从存储器中读取指令。
  2.  **译码 (Decode)**：识别指令的操作码，准备控制信号。
  3.  **执行 (Execute)**：进行 ALU 运算、移位或访问存储器。

- ARM9 五级流水线
  1.  **取指 (Fetch)**：从内存获取指令。
  2.  **译码 (Decode)**：翻译指令，读取寄存器操作数。
  3.  **执行 (Execute)**：ALU 运算或地址计算。
  4.  **存储器访问**：访问数据存储器
  5.  **寄存器回写**：将结果写回到寄存器堆

## 第三章：ARM指令系统

> [!tip] 考试重点：8种寻址方式
>
> 1. **立即寻址**：`MOV R0, #100`
> 2. **寄存器寻址**：`ADD R0, R1, R2`
> 3. **寄存器间接寻址**：`LDR R0, [R1]`
> 4. **基址变址寻址**：`LDR R0, [R1, #4]`
> 5. **多寄存器寻址**：`LDMIA R0, {R1-R5}`
> 6. **寄存器移位寻址**：`MOV R0, R1, LSL #2`
> 7. **相对寻址**：以PC为基址跳转。
> 8. **堆栈寻址**：`STMFD SP!, {R0-R3}` (满递减栈)。

### 8种寻址方式

#### 1. 立即寻址 (Immediate Addressing)

操作数直接包含在指令中，紧跟在操作码后面。

- **特点**：取指时操作数也一起被取出。
- **格式**：立即数前加 `#`。
- **示例**：
  ```
  MOV R0, #0x3F    ; 将立即数 0x3F 传送到 R0
  ADD R0, R0, #1   ; R0 = R0 + 1
  ```

#### 2. 寄存器寻址 (Register Addressing)

操作数存放在寄存器中，指令中给出的是寄存器编号。

- **特点**：执行速度最快，因为不需要访问内存。
- **示例**：
  ```
  ADD R0, R1, R2   ; R0 = R1 + R2
  MOV R1, R2       ; 将 R2 的值传送到 R1
  ```

#### 3. 寄存器移位寻址 (Register Shift Addressing)

这是 ARM 的独创特色。在对寄存器操作之前，先对其进行移位操作。

- **移位类型**：LSL（逻辑左移）、LSR（逻辑右移）、ASR（算术右移）、ROR（循环右移）、RRX（带进位扩展循环右移）。
- **示例**：
  ```
  MOV R0, R1, LSL #2     ; R0 = R1 * 4 (左移2位)
  ADD R0, R1, R2, LSR #3  ; R0 = R1 + (R2 / 8)
  ```

#### 4. 寄存器间接寻址 (Register Indirect Addressing)

寄存器中存放的是操作数在内存中的**有效地址**。

- **格式**：寄存器用 `[]` 括起来。
- **示例**：
  ```
  LDR R0, [R1]     ; 将以 R1 的值为地址的存储器中的数据传送到 R0
  STR R0, [R1]     ; 将 R0 的内容存储到以 R1 的值为地址的存储器中
  ```

#### 5. 基址变址寻址 (Base-Indexed Addressing)

将基址寄存器的内容与偏移量相加，得到最终的有效地址。

- **偏移量**：可以是立即数或寄存器。
- **常见形式**：
  1. **前索引**：`LDR R0, [R1, #4]`（R0 = \[R1+4\]）
  2. **自动索引**：`LDR R0, [R1, #4]!`（R0 = \[R1+4\]，且 R1 = R1 + 4）
  3. **后索引**：`LDR R0, [R1], #4`（R0 = \[R1\]，且 R1 = R1 + 4）
- **示例**：
  ```
  LDR R0, [R1, R2] ; R0 = [R1 + R2]
  ```

#### 6. 多寄存器寻址 (Multiple Register Addressing)

一条指令可以完成多个寄存器与连续内存单元之间的数据传输。

- **助记符后缀**：IA (After)、IB (Before)、DA (After)、DB (Before)。
- **示例**：
  ```
  LDMIA R0, {R1, R2, R5} ; 从 R0 开始的地址读取 3 个字到 R1, R2, R5
  STMIA R0!, {R1-R3}      ; 将 R1-R3 的内容存入 R0 指向的地址，R0 自动增加
  ```

#### 7. 堆栈寻址 (Stack Addressing)

专门用于堆栈操作（SP 寄存器）。ARM 支持四种类型的堆栈，但 **FD（满递减）** 是最常用的。

- **四种类型**：
  - **FA** (Full Ascending)：满递增
  - **FD** (Full Descending)：满递减（ARM 默认/常用）
  - **EA** (Empty Ascending)：空递增
  - **ED** (Empty Descending)：空递减
- **示例**：

```ARM assembler
    STMFD SP!, {R0-R3, LR} ; 将 R0-R3 和 LR 入栈（保护现场）
    LDMFD SP!, {R0-R3, PC} ; 将栈内容恢复到 R0-R3 和 PC（恢复现场并返回）
```

#### 8. 相对寻址 (Relative Addressing)

以程序计数器 **PC** 的当前值为基址，指令中的标号（Label）作为偏移量。

- **用途**：常用于分支跳转。
- **示例**：
  ```
  B  loop      ; 跳转到 loop 标号处
  BL sub_func  ; 跳转到子程序 sub_func，并保存返回地址到 LR
  ```

### 常用核心指令

- **加载/存储**：`LDR` (读内存), `STR` (写内存)。
- **跳转**：
  - `B`：简单跳转。
  - `BL`：带返回的跳转（用于函数调用）。
  - `BX`：带状态切换的跳转（ARM/Thumb切换）。

- **状态寄存器访问**：`MRS` (读PSR), `MSR` (写PSR)。

## 第四章：ARM汇编语言及C语言程序设计基础

### 1. 汇编伪操作

- `AREA`：定义代码段或数据段。
- `ENTRY`：指定程序执行入口。
- `EXPORT` / `IMPORT`：声明全局标号 / 引用外部标号。

### 2. ATPCS 寄存器使用规则

> [!important] 函数调用约定
>
> - **R0 - R3**：用于传递函数参数，R0 存放返回值。
> - **R4 - R11**：用于保存局部变量。
> - **R13 (SP)**：栈指针，必须指向有效内存。
> - **R14 (LR)**：保存返回地址。

## 第五章：嵌入式系统硬件技术基础

### 1. 存储器系统分类

**按存储方式分类**

- 随机存储器：任何存储单元的内容都能被随机存取
- 顺序存储器：只能按照某种顺序来存取

**按读写功能分类**

- 随机读写存储器：能读出能写入
- 只读存储器：只能读不能写

### 2. 最小系统组成

- 微处理器 (CPU)
- 时钟电路 (晶振)
- 电源电路
- 复位电路
  ![嵌入式系统硬件组成](/blogs/obsn-qian-ru-shi-xi-tong-qi-mo-fu-xi-bi-ji/嵌入式系统硬件组成.jpg)

### 3. 并行总线

并行总线由数据总线、地址总线、控制总线组成。

- **数据总线**：用于传输数据，双向三态总线，可以把CPU的数据传输到存储器或I/O接口等其他部件，也可以将其他部件的数据传输到CPU。
- **地址总线**：用于传输地址，单向三态总线，只能从CPU传向外部存储器或I/O端口
- **控制总线**：用于传输控制信号和时序信号

### 4. 串行总线

串行总线是将数据按比特逐位发送到目的端，按照数据与时钟是否同步，可将串行总线分为同步串行总线（I2C、SPI）和异步串行总线（USB、RS-422、RS-485）。

- **I2C**：
  - 2线制（SDA, SCL）。
  - 半双工，支持多主多从。
- **SPI**：
  - 4线制（MOSI, MISO, SCLK, CS）。
  - 全双工，速度比I2C快。
- **USB**：支持热插拔、分层拓扑。
