#### 一、选择题

1. **D** (ARM 是 RISC 架构，不是 CISC。)

2. **A** (R15 是 PC 程序计数器，R14 是 LR 链接寄存器，R13 是 SP 堆栈指针。)

3. **B** (小端模式：低位字节存放在低地址。0x87654321 的最低位字节是 0x21，存放在 1000H。)

4. **B** (EOR 是异或。R1=...78, 0x78 XOR 0xFF = 0x87。高位不变。)

5. **D** (地址计算：`R1 + (R2 >> 1)`。$R2=0x11=17$，右移1位是8 ($0x08$)。$0x1000 + 0x08 = 0x1008$。)

#### 二、填空题

1. **(1) 硬件**，**(2) 应用** (通常模型为硬件层、驱动/软件层、中间件、应用层。)

2. **(3) 译码**，**(4) 访存**，**(5) 写回** (经典的5级流水线：Fetch, Decode, Execute, Memory, Writeback。)

3. **(6) 37** (总数37)，**(7) 状态寄存器** (通常指CPSR+SPSR)，**(8) 16** (R0-R15 是程序员可视的，或者填 **16**。) _注：如果特指任意模式下物理上完全相同的寄存器，答案可能是R0-R7等，但通常填16指通用寄存器组_。

4. **(9) 哈佛** (Harvard)

5. **(10) 4** (32位系统寻址4GB)，**(11) 大端**，**(12) 小端** (顺序可互换)，**(13) 大端** (Big Endian 是高字节存低地址。)

6. **(14) 算术右移**，**(15) 1** (或者填 **符号位**，即若原数为负，补1)。

#### 三、判断题

1. **×** (制造/授权主要是 ARM 公司，Intel 早期生产 StrongARM 但不是 ARM 的拥有者/主要制造者。)

2. **×** (Cortex-A 是高性能应用处理器，Cortex-M 才是主要用于低功耗嵌入式控制。)

3. **×** (这是冯诺依曼结构的定义，哈佛结构是分开的。)

4. **×** (ARM9 采用的是哈佛结构。)

5. **√** (MOV指令的立即数必须符合8位图经过偶数位循环右移得到。0x9000 可以由 0x90 循环右移得到，是合法的立即数。)

#### 四、程序分析题

1. 指令注释：

(1) 将 R1 和 R2 相加，结果存入 R0。

(2) 将 R0 指向的地址处的连续数据加载到 R1, R2, R3, R4 中，每次加载后地址增加。

(3) 如果大于 (GT)，则比较 R13 和 (R7 左移 2 位) 的值。

(4) 将 R3 循环右移 R6 位后，与 R15 进行异或运算，结果存入 R0，并更新状态标志位。

(5) 乘加指令：$R10 = R2 \times R1 + R5$。

(6) 无符号长乘：$R1:R0 = R2 \times R3$ (R0存低32位，R1存高32位)。

2. 循环计算：

代码逻辑：

- 初始：R0=5, R1=3

- 循环：R2 = R0 + R1，然后 R0 自减 1，直到 R0=0 退出。

- 过程推演：
  - 第1次：R2 = 5+3=8, R0=4

  - 第2次：R2 = 4+3=7, R0=3

  - ...

  - 最后一次 (R0=1时)：R2 = 1+3=**4**, R0=0 (循环结束)

- 结果：
  - **(7) 0**

  - **(8) 3** (R1 始终未变)

  - **(9) 4** (最后一次赋值的值)

#### 五、简答题

**1. 什么是嵌入式系统，嵌入式系统由哪些组成部分？**

- **定义：** 嵌入式系统是以应用为中心，以计算机技术为基础，软硬件可裁剪，适应应用系统对功能、可靠性、成本、体积、功耗严格要求的专用计算机系统。

- **组成部分：**
  1. **嵌入式硬件：** 包括嵌入式处理器 (CPU)、存储器 (Flash, RAM)、外围设备 (I/O接口) 等。

  2. **嵌入式软件：** 包括嵌入式操作系统 (RTOS/Linux)、板级支持包 (BSP/驱动)、中间件和上层应用程序。

**2. 程序状态寄存器 (CPSR) 的控制位和条件标志代表的含义。**

- **条件标志位 (高4位)：**
  - **N (Negative):** 运算结果为负时置1。

  - **Z (Zero):** 运算结果为零时置1。

  - **C (Carry):** 加法产生进位或减法产生借位时置1，移位操作中最后移出的位。

  - **V (Overflow):** 符号数运算发生溢出时置1。

- **控制位 (低8位)：**
  - **I:** IRQ 中断禁止位 (1禁止)。

  - **F:** FIQ 快速中断禁止位 (1禁止)。

  - **T:** 状态位 (1为Thumb状态，0为ARM状态)。

  - **M[4:0]:** 模式位，决定处理器处于哪种工作模式 (如User, SVC, IRQ等)。

#### 编程题

**题目：计算 $S=\sum_{n=1}^{N} n$**

代码段

```
    AREA Sum_Example, CODE, READONLY
    ENTRY

    ; 假设 N 的值设定为 100，存放在 R0 中
    MOV R0, #100      ; N = 100
    MOV R1, #0        ; S = 0 (累加和结果)
    MOV R2, #1        ; n = 1 (计数器)

Loop
    ADD R1, R1, R2    ; S = S + n
    ADD R2, R2, #1    ; n = n + 1
    CMP R2, R0        ; 比较 n 和 N
    BLE Loop          ; 如果 n <= N，跳转回 Loop 继续执行

Stop
    B Stop            ; 死循环结束程序

    END
```

_(注：也可以使用倒序相加，代码更精简)_

代码段

```
    MOV R0, #100      ; N
    MOV R1, #0        ; S
Loop
    ADD R1, R1, R0    ; S = S + N
    SUBS R0, R0, #1   ; N = N - 1，并更新标志位
    BNE Loop          ; 如果 R0 不等于 0，继续循环
```
