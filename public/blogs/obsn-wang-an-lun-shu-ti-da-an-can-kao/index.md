# 网安论述题答案参考

### 1. 网络空间安全具有哪些特点？

1. 网络攻击目标发生变化
   - **趋势：** 攻击目标由过去的攻击个人、单机、网站等，向攻击**政府网站、关键基础设施、全球侵入**发展。
   - **现状：** 针对关键基础设施（如工业控制系统）的高强度精准网络攻击频次和强度持续增长。
2. 网络安全风险由漏洞、后门向供应链、产业短板发展
   - **核心问题：** 核心技术受制于人是最大的隐患。
   - **趋势：** 风险不再仅限于软件漏洞或后门，而是延伸到了**供应链安全**和**核心技术/产业短板**。发达国家利用技术垄断和供应链优势（如芯片、操作系统等），形成了“卡脖子”的技术壁垒和封锁（文中提到了Windows XP停服及对中兴、华为的制裁作为例子）。
3. 网络安全焦点由物理打击破坏向数据掠夺和资源控制转变
   - **核心观点：** 数据已成为继水、电、油、核之后的又一重要**国家战略资源**。
   - **趋势：** 网络空间的主权体现为对数据的掌控。西方发达国家利用技术优势，正从全球源源不断地搜取各类数据，大肆进行数据掠夺和资源控制，以谋取长远战略优势
4. 网络攻击威胁由无组织的黑客行为向有组织的网军行动转变
   - **趋势：** 网络攻击不再是过去“散兵游勇”式的个人黑客行为，而是演变为**以网络作战为目的的网军**。
   - **现状：** 全球多国已建立网军，研发网络攻击武器，建立漏洞库和网络靶场，并频繁组织大规模攻防演习。这加剧了网络空间的军备竞赛，使世界和平受到新挑战。

### 2. 信息安全三准则及应用分析

- **三准则 (CIA)**：**保密性**（Confidentiality）、**完整性**（Integrity）、**可用性**（Availability）。
- **原则上只有发送方与接收方能访问**：指的是**保密性**。
- **能发现篡改行为**：指的是**完整性**。
- **哈希表比较配置文件情景**：体现了**完整性**准则。通过对比哈希值，可以确信配置文件的内容是否被非法更改。

### 3. 漏洞产生的原因有哪些？

漏洞是信息系统的某种缺陷，其产生原因包括：

- **程序设计缺陷**：如代码编写时缺乏对输入的过滤。
- **环境因素**：寄生在特定操作系统或应用流程中。
- **配置不当**：如系统默认账号未修改、权限设置过宽。
- **人为因素**：虽然漏洞不一定都是故意引入的，但人的疏忽是主要诱因。

### 4. 身边发生的网络安全实践、原因及规避方案

- **案例1：SQL注入攻击**
  - **原因**：程序对用户输入的敏感字符（如 `and`, `or`, `--`）缺乏过滤。
  - **规避方案**：对用户输入进行严格过滤或使用预编译语句。

- **案例2：社会工程学诈骗**
  - **原因**：攻击者利用获取的个人信息进行伪装和欺骗。
  - **规避方案**：提高安全意识，不点击可疑链接，多渠道确认对方身份。

- **案例3：弱密码破解**
  - **原因**：用户使用简单的字符（如 `qwerty`, `admin@123`）作为口令。
  - **规避方案**：增加密码复杂度（包含大小写字母、数字和特殊字符。

### 5. 计算机病毒与恶意代码

- **计算机病毒**：一种具有**可感染性**和**依附性**的恶意程序，能将自身代码注入到其他文件或引导区中执行。
- **恶意代码**：指故意编制或在程序中插入的、具有破坏计算机功能或毁坏数据、影响计算机使用、且能够自我复制的一组计算机指令或程序代码 。包含病毒、蠕虫、木马等 。
- **攻击过程**：通常分为侵入阶段、隐藏阶段、触发阶段、执行/破坏阶段 。

### 6. 数据恢复顺序分析

- **场景描述**：1个全量备份 (ALLHNan) + 3个连续增量备份 (ZhZhou, Lyang, Nyang) 。
- **恢复顺序**：必须先恢复全量备份，然后再按时间顺序依次恢复增量备份。
  - **顺序**：`ALLHNan` → `ZhZhou` → `Lyang` → `Nyang`。

### 7. 网络安全扫描技术的作用

- **发现安全漏洞**：检测目标主机是否存在已知的安全弱点。
- **资产普查**：识别网络中活跃的主机和开放的端口。
- **预防攻击**：在攻击者利用漏洞前，帮助管理员完善系统安全库并修补漏洞。

### 8. 监听避免、防扫描及溯源措施

- **避免监听最好的方法**：**数据加密**。建立安全的网络通信（如使用 VPN 或加密协议） 。

- **防止扫描的办法**：使用**防火墙**进行访问控制；关闭不必要的服务和端口；使用 IDS（入侵检测系统）监测异常扫描行为。
- **确认攻击来源的措施**：查看并分析 Web 服务器及操作系统的**日志文件**；通过流量分析工具（如 Wireshark）追踪数据包。

### 9. 敏感字符检测与 SQL 注入防御

- **原因分析**：用户提交 and, or, --, = 等字符，表明系统极可能正遭受 **SQL 注入攻击** 。
- **防护措施**：
  - 对用户的所有输入进行严格过滤 。
  - 使用参数化查询（预编译语句）。
  - 部署 IDS（入侵检测系统）或 WAF（Web 应用防火墙）进行实时监控。

### 10. 系统漏洞定义与防范

- **定义**：存在于信息系统中的某种缺陷，本身不具破坏性，但可被利用导致损失 。
- **产生原因**：操作系统复杂度高、设计时的逻辑错误、环境限制等。
- **防范措施**：及时安装安全补丁；关闭不必要的系统功能；使用漏洞扫描工具定期自检 。

### 11. 主动攻击与被动攻击

- **主动攻击**：攻击者访问并试图改变系统资源或影响其运作。
  - **例子**：拒绝服务攻击 (DoS) 40404040、SQL 注入、假冒地址的欺骗攻击 。
- **被动攻击**：主要是收集信息而不干扰系统正常运行。
  - **例子**：**网络窃听** 、**流量分析**（使用 Wireshark 等工具分析数据包）、通过扫描获取服务器信息。

### 12. 对称密码和非对称密码的区别和应用

| **特性**       | **对称密码**                  | **非对称密码**               |
| -------------- | ----------------------------- | ---------------------------- |
| **密钥数量**   | 单个密钥（共享密钥）          | 密钥对（公钥和私钥）         |
| **安全性依赖** | 密钥的秘密传输与保管          | 仅需保密私钥，公钥可公开     |
| **性能**       | 加密速度快，适合大数据量      | 加密算法复杂，速度慢         |
| **典型算法**   | **DES**, AES                  | **RSA**, DSA                 |
| **应用场景**   | 数据的批量加密（如 VPN 通信） | 密钥分发、数字签名、身份识别 |
