## 嵌入式系统基础

### 1. 简单描述嵌入式系统的定义及其特点。

**答：**

- **嵌入式系统定义：** 嵌入式系统是以应用为中心，以计算机技术为基础，软件、硬件可剪裁，适应应用系统对功能、可靠性、成本、体积和功耗严格要求的专用计算机系统，通常嵌入到其他设备中，完成特定的功能 。

- **嵌入式的特点：**
  1. **专用性强：** 针对特定任务设计，功能单一 。
  2. **软硬件可裁剪：** 根据需求调整硬件配置与软件功能 。
  3. **实时性：** 嵌入式系统需在规定时间内完成任务 。
  4. **可靠性高：** 在特定环境下长期运行，稳定不出故障 。
  5. **其他：** 功耗低、体积小、功能强、灵活方便、集成度高 。

### 2. 简单描述嵌入式系统的组成结构及其各部分的功能与作用。

**答：** 嵌入式系统由硬件层、中间层、软件层和功能层构成 。

- **硬件层：** 由嵌入式微处理器、存储器、通用外部接口组成 。
  - **功能：** 承担系统的核心运算、逻辑判断与任务控制，是处理数据与执行指令的“大脑”，决定系统的运算能力，并存放可执行代码与数据 。
- **中间层（硬件抽象层）：** 将系统上层软件与底层硬件分离开，使底层驱动程序与硬件无关 。
  - **功能：** 包含底层硬件初始化、数据输入/输出操作、硬件配置功能，使开发人员无须关心底层硬件细节 。
- **软件层：** 由嵌入式操作系统、文件系统、图形用户接口、网络系统及通用组件模块组成 。
  - **功能：** 负责软硬件资源的分配、任务调度、内存管理、中断处理、控制并协调并发活动 。
- **功能层（应用软件层）：** 基于实时系统开发的应用程序组成 。
  - **功能：** 实现对被控对象的控制功能，面向受控对象和用户，提供友好的人机界面，方便用户操作 。

## ARM 体系架构

### 1. 简单描述流水线的定义，并分析 ARM7 和 ARM9 采用的体系架构，且说明采用的流水线技术。

**答：**

- **流水线定义：** 流水线技术指将一个重复的时序过程分解成为若干子过程，每个子过程在专用功能段上同时执行 。
- **ARM7：** 采用冯·诺依曼结构。使用三级流水线技术（取指 $\rightarrow$ 译码 $\rightarrow$ 执行）。
- **ARM9：** 采用哈佛结构。使用五级流水线技术（取指 $\rightarrow$ 译码 $\rightarrow$ 执行 $\rightarrow$ 存储器访问 $\rightarrow$ 寄存器回写）。

### 2. 简单描述 ARM 体系结构版本的命名规则，说明 ARM9TDMI 的含义。

**答：**

- **命名规则：** 以数字表示内核的系列号，以字母表示内核所支持的额外功能 。
  **ARM{x}{y}{z}{T}{D}{M}{I}{E}{J}{F}{-S}**
- **x**：系列代号（如 7, 9, 10, 11）。
- **y**：存储管理/保护单元。
- **z**：内含Cache 。
- T：支持 16 位 Thumb 指令集
- D：支持 JTAG 调试
- M：增强型长乘法器 (Fast Multiplier)
- I：带有嵌入式追踪宏单元ETM，用来设置断点和观察点的调试硬件
- E：增强型 DSP 指令
- J：支持 Java 加速技术
- F：具备硬件浮点运算单元
- S：可综合版本
- **ARM9TDMI 含义：**
  - **9：** 表示内核系列号 。
  - **T：** 表示支持 16 位的 Thumb 指令集 。
  - **D：** 支持 JTAG 调试 。
  - **M：** 指支持快速乘法操作 。
  - **I：** 用来设置断点和观察点调试硬件（EmbeddedICE）。

### 3. 简单描述 ARM 处理器的寄存器分类及各自的功能，详细说明 CPSR 寄存器状态位的作用。

**答：**

- **寄存器分类：** 共有 37 个 32 位寄存器（31 个通用寄存器，6 个状态寄存器）。
  - **$R_0 - R_7$：** 未分组寄存器，无特殊用途，用于存放临时数据 。
  - **$R_8 - R_{12}$：** 分组寄存器。在 FIQ 模式下访问其私有寄存器，其他模式访问用户模式寄存器 。
  - **$R_{13}$ (SP)：** 堆栈指针，用于保存堆栈地址 。
  - **$R_{14}$ (LR)：** 连接寄存器，跳转时保存返回地址 。
  - **$R_{15}$ (PC)：** 程序计数器，保存将要执行的指令地址 。

- **CPSR（当前程序状态寄存器）状态位作用：**
  - **条件码标志位：**
    - **N (Negative)：** 负标志位，$N=1$ 表示运算结果为负数 。
    - **Z (Zero)：** 零标志位，$Z=1$ 表示运算结果为零 。
    - **C (Carry)：** 进位/借位标志位。加法运算产生进位时 $C=1$，减法运算无借位时 $C=1$（有借位 $C=0$）。
    - **V (Overflow)：** 溢出标志位，带符号数运算时 $V=1$ 表示符号位溢出 。
  - **控制位：**
    - **I：** $I=1$ 时禁止 IRQ 中断 。
    - **F：** $F=1$ 时禁止 FIQ 中断 。
    - **T：** 反映处理器的运行状态（ARM/Thumb）。
    - **M (Mode bits)：** 决定处理器的运行模式 。
  - **保留位：** 用于 ARM 版本的扩展 。

## 硬件接口与总线

### 1. 简单描述嵌入式系统的硬件组成及其各部分的功能与作用。

**答：**

- **嵌入式处理器：** 系统的核心，负责数据处理与控制 。
- **存储器：** 存放数据和程序 。
- **外围设备：** 决定不同领域的嵌入式系统的独特功能。
- **输入输出接口：** 传递信息的公用通道 。
- **总线：** 各功能部件间信息传输的路径 。
- **外围电路：** 提供稳定的电流与电压 。
  ![嵌入式系统硬件组成](/blogs/obsn-jian-da-ti/嵌入式系统硬件组成.jpg)

### 2. 简单描述常用的串行总线，并详细比较 I2C 总线和 SPI 总线的异同。

**答：**

- **串行总线：** 将数据按比特逐位发送。包括同步串行总线（$I^2C$、SPI）和异步串行总线（USB、RS-422/232） 。

- **I2C 与 SPI 的异同：**
  - **相同点：** 均为同步串行总线；由主设备提供时钟信号；适用于短距离、低功耗通信；均可采用主从结构 。
  - **不同点：**

| **特性**       | **I2C**                        | **SPI**                                        |
| -------------- | ------------------------------ | ---------------------------------------------- |
| **信号线数量** | 2 根（SDA 数据线, SCL 时钟线） | 4 根（SDI 输入, SDO 输出, SCLK 时钟, CS 片选） |
| **通信方式**   | 同步半双工                     | 同步全双工                                     |
| **结构**       | 多主多从                       | 一主多从                                       |
| **寻址方式**   | 软件寻址                       | 硬件寻址（通过片选线）                         |

### 3. 简单描述 RS232 协议及其帧格式（字符传送格式）。

**答：**

- **RS232 协议：** 异步串行通信的物理层协议，定义了数据终端设备 (DTE) 和数据通信设备 (DCE) 之间的电气、机械和功能特性 。
- **电气特性：** 采用负逻辑电平，即逻辑“1”用 $-15V \sim -3V$ 表示，逻辑“0”用 $+3V \sim +15V$ 表示 。
- **帧格式（字符传送格式）：**
  1. **空闲位：** 处于逻辑“1”状态 。
  2. **起始位：** 逻辑“0”，表示字符传输开始 。
  3. **数据位：** 紧跟起始位，通常为 5~8 位 。
  4. **奇偶校验位：** 可选，用于检错 。
  5. **停止位：** 逻辑“1”，表示一个字符传送结束 。
